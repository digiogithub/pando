# Plan de Correcciones TUI Pando - 2026-03-08

## Problemas Identificados

### 1. Error Copilot API: "Unrecognized request argument supplied: reasoning_effort"
- **Archivo**: `internal/llm/provider/copilot.go:285-296`
- **Causa raíz**: El método `preparedParams()` envía `reasoning_effort` para TODOS los modelos con `CanReason: true`. Sin embargo, la API de GitHub Copilot NO soporta este parámetro para todos los modelos (ej: GPT-5.4, Claude models vía Copilot proxy).
- **Modelos afectados**: GPT-4.1, o1, o3-mini, o4-mini, Claude 3.7 Thought, GPT-5.4, Claude Opus 4 - todos tienen `CanReason: true`
- La API de Copilot solo soporta `reasoning_effort` para modelos OpenAI o-series (o1, o3-mini, o4-mini)

### 2. Panel de Ayuda sin scroll y contenido cortado
- **Archivo**: `internal/tui/components/dialog/help.go`
- **Causa raíz**: El componente `helpCmp` renderiza todo el contenido directamente sin usar un `viewport.Model` de bubbles. Cuando el contenido excede la altura de la pantalla, se corta.
- No hay altura máxima (fixed height), no hay scroll, no hay manejo de teclas de navegación (j/k, pgup/pgdn)

### 3. Colores de fondo inconsistentes
- **Archivos**: `help.go`, `styles.go`, overlay system
- **Causa raíz**: 
  - En `renderSection()`, los estilos `helpKeyStyle` y `helpDescStyle` usan `Regular()` (sin background por defecto) con `.Background(t.Background())`, pero el espacio entre elementos no tiene background aplicado
  - El `baseStyle.Render(strings.Repeat(" ", ...))` en línea 126 usa `BaseStyle()` que tiene background, pero las uniones horizontales con `lipgloss.JoinHorizontal` no rellenan el fondo uniformemente
  - La sección "Navigation" y las siguientes muestran un color de fondo diferente porque el contenido no llena el ancho completo del panel

### 4. Falta Ctrl+i para editar archivos y Ctrl+s para guardar
- **Archivos**: `internal/tui/keys.go`, `internal/tui/components/editor/tabs.go`, `internal/tui/tui.go`
- **Estado actual**: No existen keybindings para editar archivos en un editor externo (Ctrl+i) ni para guardar (Ctrl+s está asignado a "switch session")
- Ctrl+s conflicta con switch session - necesita reasignación
